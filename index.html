Crush 学习教程

从零开始掌握 Charmbracelet 出品的终端 AI 编程助手 Crush，Go 语言编写，10MB 大小，支持多种 LLM，LSP 深度集成，极简美学。

Crush：Charm 出品的终端 AI 编程助手

⬇️ 安装与环境 入门

系统要求

系统要求推荐终端
macOSmacOS 11+Ghostty, WezTerm, iTerm2, Kitty
Linuxglibc 2.31+WezTerm, Alacritty, Ghostty, Kitty
WindowsWindows 10+Windows Terminal (推荐 WSL)
💡 提示：Windows 用户建议使用 WSL（Windows Subsystem for Linux）以获得最佳体验。

安装方式

bash — 推荐方式
# 一键安装（自动检测系统架构） curl -fsSL https://charm.sh/install/crush | bash # 安装后验证 crush --version
bash — macOS/Linux
# Homebrew 安装 brew install charmbracelet/tap/crush # 或使用官方 tap brew tap charmbracelet/tap brew install crush
bash — Go 安装
# Go install（需要 Go 1.21+） go install github.com/charmbracelet/crush@latest # 确保 $GOPATH/bin 在 PATH 中 export PATH=$PATH:$(go env GOPATH)/bin
下载地址
# GitHub Releases 页面 https://github.com/charmbracelet/crush/releases # 下载对应平台的二进制文件后 chmod +x crush sudo mv crush /usr/local/bin/

验证安装

bash
# 检查版本 crush --version # 启动 Crush crush # 指定工作目录 crush -c /path/to/project # 查看帮助 crush --help

🚀 5分钟上手 入门

第一步：配置 API 密钥

Crush 需要配置 AI 模型的 API 密钥才能工作。支持多种 LLM 提供商：

bash — 设置环境变量
# Anthropic Claude（推荐） export ANTHROPIC_API_KEY="sk-ant-..." # OpenAI GPT export OPENAI_API_KEY="sk-..." # Google Gemini export GEMINI_API_KEY="AIza..." # 将环境变量添加到 shell 配置文件 echo 'export ANTHROPIC_API_KEY="sk-ant-..."' >> ~/.zshrc source ~/.zshrc
💡 建议将环境变量添加到 shell 配置文件：~/.zshrc 或 ~/.bashrc，这样每次打开终端都会自动加载。

第二步：启动 Crush

bash
# 进入项目目录 cd /path/to/your/project # 启动 Crush crush

🆕 首次启动

Crush 首次启动时会：

  • 自动检测项目类型和语言
  • 通过 LSP 分析项目结构
  • 加载项目上下文（文件树、依赖等）
  • 显示欢迎界面和快捷键提示

第三步：开始对话

示例对话
# 理解代码 "解释一下这个项目的路由处理逻辑" # 修复 Bug "修复这个错误：nil pointer dereference in main.go:42" # 添加功能 "给 UserService 添加一个批量删除用户的方法" # 生成测试 "为 auth/handler.go 中的 Login 函数生成单元测试" # 重构代码 "将这个函数拆分成更小的辅助函数"

第四步：切换模型

模型切换
# 快捷键切换模型 Ctrl+O → 打开模型选择对话框 # 命令行指定 crush --model claude-sonnet-4-20250514 crush --model gpt-4o crush --model gemini-2.5-pro

对话技巧

💡 高效对话要点

  • 具体明确：描述清楚你要做什么，而不是模糊的需求
  • 提供上下文：告诉 Crush 相关的文件名和函数名
  • 分步进行：复杂任务拆分成小步骤，逐步确认
  • 善用 /undo：如果结果不满意，随时撤销重试
  • 检查代码：AI 生成的代码建议先审查再应用

🔄 工作模式 入门

双模式详解

维度🧠 Plan（计划模式）🔨 Build（构建模式）
权限只读可读写
文件操作只能查看可以修改、创建、删除
命令执行不能执行 shell可以执行 shell 命令
适用场景理解代码、制定计划实现功能、修复 Bug
安全性高（无副作用）中（需要确认操作）
切换方式按 Tab 键切换

推荐工作流

推荐工作流：先计划后构建，确保安全可靠
💡 最佳实践：复杂任务先用 Plan 模式分析，确认方案后再切换到 Build 模式执行。如果结果不满意，使用 /undo 撤销。

🔧 内置工具 中级

Crush 内置了丰富的工具，让 AI 能够直接操作你的项目：

📁 文件操作

读取、写入、编辑、搜索项目文件。支持 glob 模式匹配和正则搜索。

💻 Shell 命令

在 Build 模式下执行 shell 命令。运行测试、安装依赖、编译构建。

🔍 LSP 集成

通过语言服务器协议获取代码智能提示：类型信息、引用跳转、符号搜索。

🌐 网络请求

获取网页内容，搜索在线资源。适合需要查阅文档的场景。

📋 Git 操作

查看 Git 状态、提交历史、差异比较。了解代码变更上下文。

🤖 子代理

启动子代理处理特定任务，如文件搜索、代码分析等。支持并行执行。

工具使用示例

对话示例 — 让 Crush 使用工具
# 让 Crush 搜索代码 "搜索所有包含 'TODO' 的注释" # 让 Crush 运行命令 "运行 go test ./... 并告诉我结果" # 让 Crush 读取文件 "读取 config.yaml 并解释配置项" # 让 Crush 修改文件 "在 main.go 中添加一个健康检查端点" # 让 Crush 查看 Git 状态 "查看最近的提交记录，总结改动内容"

🌳 Charm 生态 中级

Crush 是 Charmbracelet 生态的一部分，与以下工具完美配合：

🖥️ Ghostty — 现代终端模拟器▶

Ghostty 是 Charmbracelet 团队打造的 GPU 加速终端模拟器，与 Crush 配合获得最佳体验。

  • GPU 渲染：流畅的终端渲染，支持高 DPI
  • 内置主题：与 Crush 深色主题完美匹配
  • 跨平台：macOS、Linux、Windows 全支持
  • 轻量快速：启动速度快，资源占用低
bash — 安装 Ghostty
# macOS brew install --cask ghostty # Linux sudo apt install ghostty
🎨 Lip Gloss — 终端样式库▶

Go 语言的终端样式库，Crush 的 TUI 界面就是基于 Lip Gloss 构建的。

  • 声明式样式定义
  • 自动边距和填充
  • 颜色和样式支持
  • 响应式布局
🫧 Bubble Tea — TUI 框架▶

Go 语言最流行的 TUI（终端用户界面）框架，Crush 的交互界面基于 Bubble Tea 构建。

  • 基于 Elm 架构
  • 丰富的内置组件
  • 键盘和鼠标事件支持
  • 跨平台兼容
🐝 Huh — 表单库▶

Charmbracelet 的表单和交互组件库，用于构建 Crush 中的交互式对话框。

🔒 隐私安全 高级

隐私保护承诺

🛡️ Crush 的隐私原则

  • 本地存储：所有对话和历史记录仅保存在本地 SQLite 数据库
  • 代码不离开本地：除非你使用云端模型 API
  • 无遥测：默认不收集任何使用数据
  • 开源透明：MIT 开源协议，代码完全公开

安全最佳实践

API 密钥管理▶
bash
# ✅ 推荐：使用环境变量 export ANTHROPIC_API_KEY="sk-ant-..." # ❌ 避免：硬编码在配置文件中 # ✅ 使用 .env 文件（添加到 .gitignore） echo "ANTHROPIC_API_KEY=sk-ant-..." >> .env echo ".env" >> .gitignore
敏感项目处理▶
  • 使用 Plan 模式 查看敏感文件，避免意外修改
  • 使用 Ollama 本地模型，代码完全不离开本地
  • 将 .crush/ 目录添加到 .gitignore

Crush 和 OpenCode 是什么关系？▶

Crush 是 OpenCode 的精神续作。原 OpenCode 仓库已于 2025 年归档，Charmbracelet 团队接手并重新打造了 Crush。

Crush 继承了 OpenCode 的核心理念，但用 Go 重写，更加轻量快速，并且深度集成了 Charm 生态。

如何选择模型？▶
  • 编程推荐：Claude Sonnet 4（性价比最高）
  • 最强推理：Claude Opus 4 或 o3
  • 免费方案：Ollama + CodeLlama（完全离线）
  • 速度优先：Groq + Llama 3.3 或 Claude Haiku
支持哪些编程语言？▶

Crush 通过 LSP 支持所有主流编程语言：

Go、Python、TypeScript/JavaScript、Rust、Java、C/C++、Ruby、PHP 等。

LSP 会自动检测项目语言并启用对应的语言服务器。

Crush 占用多少资源？▶
  • 二进制大小：约 10MB
  • 内存占用：约 30-50MB（不含 LSP 服务器）
  • 启动时间：< 1 秒
  • 磁盘空间：~15MB（含本地数据库）
