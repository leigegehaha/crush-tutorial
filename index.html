⬇️ 安装与环境 入门

系统要求

系统要求推荐终端
macOSmacOS 11+Ghostty, Alacritty, iTerm2, Kitty
Linuxglibc 2.31+Ghostty, Alacritty, WezTerm, Kitty
WindowsWindows 10+Windows Terminal (推荐 WSL)
💡 提示：Crush 由 Charmbracelet 开发，与 Ghostty 终端配合使用效果最佳。

安装方式

bash — 推荐方式
# 使用官方安装脚本 curl -fsSL https://github.com/charmbracelet/crush/raw/main/install.sh | bash # 或使用 wget wget -qO- https://github.com/charmbracelet/crush/raw/main/install.sh | bash
bash — macOS/Linux
# Homebrew 安装 brew install charmbracelet/tap/crush # 或先添加 tap brew tap charmbracelet/tap brew install crush
bash — Go 安装
# Go 安装（需要 Go 1.23+） go install github.com/charmbracelet/crush@latest # Arch Linux AUR yay -S crush-bin # 或 paru -S crush-bin
bash — 手动下载
# 从 GitHub Releases 下载 # https://github.com/charmbracelet/crush/releases # macOS (Apple Silicon) curl -sLO https://github.com/charmbracelet/crush/releases/download/v0.3.0/crush_0.3.0_darwin_arm64.tar.gz tar -xzf crush_0.3.0_darwin_arm64.tar.gz sudo mv crush /usr/local/bin/ # Linux (x64) curl -sLO https://github.com/charmbracelet/crush/releases/download/v0.3.0/crush_0.3.0_linux_amd64.tar.gz tar -xzf crush_0.3.0_linux_amd64.tar.gz sudo mv crush /usr/local/bin/

验证安装

bash
# 检查版本 crush --version # 查看帮助 crush --help # 启动 Crush crush
⚠️ 注意：首次运行需要配置 API Key，请确保已准备好至少一个 LLM 提供商的 API Key。

🚀 5分钟上手 入门

第一步：配置 API Key

bash — 设置环境变量
# Anthropic Claude (推荐) export ANTHROPIC_API_KEY="your-api-key" # 或 OpenAI export OPENAI_API_KEY="your-api-key" # 或 Google Gemini export GEMINI_API_KEY="your-api-key" # 或本地模型（Ollama） # 无需 API Key，确保 Ollama 服务运行中

第二步：启动 Crush

bash
# 在项目目录中启动 cd your-project crush # 或指定工作目录 crush --cwd /path/to/project # 使用特定模型启动 crush --model claude-3.5-sonnet

第三步：开始对话

对话示例
你: 帮我分析这个项目的结构 你: 修复 src/auth.go 中的那个 bug 你: 为这个函数添加单元测试 你: 重构这个模块，提高可读性
💡 提示：Crush 会自动读取项目上下文，你可以直接提问而无需详细说明文件位置。

切换模型

在 Crush 中切换模型
# 查看当前模型 /model # 切换到 Claude 3.5 Sonnet /model claude-3.5-sonnet # 切换到 GPT-4o /model gpt-4o # 切换到本地 Ollama 模型 /model ollama:llama3.2 # 切换到 Gemini /model gemini-2.0-flash

配置文件

yaml — ~/.config/crush/crush.yml

配置文件位置

系统配置目录

Windows%APPDATA%\crush\

完整配置示例

yaml — crush.yml
# 默认模型设置 model: claude-3.5-sonnet # 默认提供商 provider: anthropic # 提供商详细配置 providers: anthropic: api_key: ${ANTHROPIC_API_KEY} base_url: https://api.anthropic.com openai: api_key: ${OPENAI_API_KEY} base_url: https://api.openai.com/v1 gemini: api_key: ${GEMINI_API_KEY} ollama: base_url: http://localhost:11434 groq: api_key: ${GROQ_API_KEY} openrouter: api_key: ${OPENROUTER_API_KEY} # 工具配置 tools: # 启用的工具 enabled: - file_read - file_write - shell_execute - web_search - http_client - git # Shell 工具配置 shell: timeout: 60s safe_commands: - git status - git diff - go test - npm test # 上下文配置 context: max_tokens: 128000 include_gitignore: true exclude_patterns: - "node_modules/**" - ".git/**" - "vendor/**" - "*.lock" # 界面配置 ui: theme: charm show_token_usage: true confirm_actions: true

🛠️ 内置工具 中级

核心工具列表

📁 文件操作工具 ▶
  • file_read — 读取文件内容，支持多种编码
  • file_write — 创建或覆盖写入文件
  • file_edit — 精确编辑文件（类似 sed）
  • file_search — 在文件中搜索内容
  • directory_list — 列出目录内容
🖥️ Shell 执行工具 ▶
  • shell_execute — 执行终端命令
  • shell_background — 后台执行长时间任务
⚠️ 安全提示：危险命令需要用户确认后才会执行。
🌐 网络工具 ▶
  • http_client — HTTP 请求（GET/POST/PUT/DELETE）
  • web_search — 网络搜索（需要配置搜索 API）
  • web_fetch — 获取网页内容
📦 Git 工具 ▶
  • git_status — 查看仓库状态
  • git_diff — 查看差异
  • git_commit — 创建提交
  • git_branch — 分支管理

工具管理命令

工具管理
# 列出所有可用工具 /tools # 启用/禁用特定工具 /tools enable shell_execute /tools disable web_search # 查看工具详情 /tools info file_write

🔧 自定义工具 高级

创建自定义工具

Crush 支持通过配置文件定义自定义工具：

yaml — ~/.config/crush/tools/custom.yml
tools: my_custom_tool: name: "My Custom Tool" description: "执行自定义操作" type: shell command: "echo 'Hello from custom tool'" timeout: 30s docker_ps: name: "Docker Status" description: "查看 Docker 容器状态" type: shell command: "docker ps -a --format 'table {{.Names}}\t{{.Status}}'" npm_scripts: name: "NPM Scripts" description: "列出可用的 npm 脚本" type: shell command: "cat package.json | jq '.scripts'"

🔒 隐私安全 高级

🏠 本地优先

支持 Ollama 等本地模型，代码和数据完全不出本地。

🔐 API Key 安全

API Key 存储在本地环境变量，不上传到任何服务器。

📝 用户确认

危险操作（删除、覆盖文件）需要用户明确确认。

🚫 无追踪

Crush 不收集任何使用数据或遥测信息。

⚠️ 注意：使用云端模型时，代码片段会发送到对应的 API 服务。如需完全隐私，请使用 Ollama 等本地模型。

❓ 常见问题 帮助

Q: Crush 与其他 AI 编程助手有什么区别？ ▶

A: Crush 由 Charmbracelet 开发，具有以下特点：

  • ✅ 完全开源（MIT 协议）
  • ✅ 支持多种 LLM 提供商，不绑定特定服务
  • ✅ 与 Charm 生态完美集成
  • ✅ 终端原生设计，符合开发者习惯
  • ✅ 支持 Ollama 等本地模型，隐私优先
Q: 如何使用 Ollama 本地模型？ ▶

A: 三步即可：

1. 安装 Ollama: curl -fsSL https://ollama.com/install.sh | sh
2. 拉取模型: ollama pull llama3.2
3. 在 Crush 中: /model ollama:llama3.2
Q: 如何保护 API Key 安全？ ▶

A: 推荐做法：

  • 使用环境变量而非明文配置
  • 将 API Key 添加到 .zshrc 或 .bashrc
  • 确保 .gitignore 包含敏感文件
  • 定期轮换 API Key
Q: Crush 支持哪些编程语言？ ▶

A: Crush 支持所有主流编程语言，包括但不限于：

Go, Rust, Python, JavaScript/TypeScript, Java, C/C++, Ruby, PHP, Swift, Kotlin, 等等

由于底层使用 LLM，语言支持取决于所使用的模型能力。
